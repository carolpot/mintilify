Use this endpoint to enrich a single contact record in real time. You can look up a contact by email address, LinkedIn URL, Lusha person ID, or a combination of first name, last name, and company. The response includes contact details such as phone numbers, email addresses, job title, location, and optional signals data.

## Endpoint

```
GET https://api.lusha.com/v2/person
```

## Authentication

<ParamField header="api_key" type="string" required>
  Your Lusha API key. Pass it as a request header named `api_key`.
</ParamField>

## Search requirements

You must supply at least one of the following combinations:

- `personId`, **or**
- `email`, **or**
- `linkedinUrl`, **or**
- `firstName` **and** `lastName` **and** (`companyName` **or** `companyDomain`)

<Info>
  Provide as many identifiers as possible to improve match accuracy. Using `personId` is the most direct way to retrieve a contact, because it uniquely identifies a record in Lusha's database.
</Info>

<Warning>
  **Unified Credits plan required for `revealEmails` and `revealPhones`**

  These parameters are only available to customers on the Unified Credits pricing plan. Calling them on any other plan returns a `403` error. When neither parameter is included, the API returns all available email addresses and phone numbers by default.
</Warning>

## Query parameters

<ParamField query="firstName" type="string">
  The first name of the person.

  Example: `Dustin`
</ParamField>

<ParamField query="lastName" type="string">
  The last name of the person.

  Example: `Moskovitz`
</ParamField>

<ParamField query="personId" type="string">
  The unique person identifier in Lusha. Using this field provides the most direct lookup.

  Example: `4183886134`
</ParamField>

<ParamField query="companyName" type="string">
  The name of the company the person works at.

  Example: `Lusha`
</ParamField>

<ParamField query="companyDomain" type="string">
  The domain name of the company.

  Example: `lusha.com`
</ParamField>

<ParamField query="email" type="string">
  The email address of the person.

  Example: `dustin@lusha.com`
</ParamField>

<ParamField query="linkedinUrl" type="string">
  The LinkedIn profile URL of the person.

  Example: `https://www.linkedin.com/in/dustin/`
</ParamField>

<ParamField query="refreshJobInfo" type="boolean">
  Set to `true` to refresh and update the job details for the person. This replaces any outdated job information with the most recent data.

  Example: `true`
</ParamField>

<ParamField query="filterBy" type="string">
  Filters results based on which contact details must be present. By default, results include contacts that have at least one contact detail available.

  Allowed values: `phoneNumbers`, `emailAddresses`
</ParamField>

<ParamField query="revealEmails" type="boolean">
  Set to `true` to retrieve only the email address of the contact. Requires the Unified Credits plan.

  Example: `true`
</ParamField>

<ParamField query="revealPhones" type="boolean">
  Set to `true` to retrieve only the phone number of the contact. Requires the Unified Credits plan.

  Example: `true`
</ParamField>

<ParamField query="signals" type="string[]">
  Array of signal types to retrieve for the contact. If no signals data is found for the specified period, the `signals` object is returned empty. Omit this parameter entirely if you do not need signals data.

  Allowed values: `allSignals`, `promotion`, `companyChange`
</ParamField>

<ParamField query="signalsStartDate" type="string">
  Start date for signal retrieval in `YYYY-MM-DD` format. Defaults to 6 months ago if not specified. Only applies when `signals` is included.

  Example: `2025-03-01`
</ParamField>

<ParamField query="partialProfile" type="boolean">
  Set to `true` to allow the API to return a partial profile when a full match is not available.

  Example: `true`
</ParamField>

### Error responses

| Status | Meaning |
|--------|---------|
| `400` | Bad request — missing or invalid parameters |
| `401` | Unauthorized — invalid or missing API key |
| `403` | Forbidden — plan restriction or access denied |
| `404` | Not found — contact does not exist |
| `429` | Too many requests — rate limit exceeded |
| `451` | Unavailable for legal reasons |
| `500` | Internal server error |

All error responses share the same shape:

<ResponseField name="statusCode" type="integer" required>
  HTTP status code.

  Example: `400`
</ResponseField>
<ResponseField name="message" type="string" required>
  Human-readable error description.

  Example: `Validation failed`
</ResponseField>
<ResponseField name="errors" type="string[]">
  Array of detailed validation error messages. Only present for validation errors.

  Example: `["entityType must be one of: contact, company"]`
</ResponseField>
