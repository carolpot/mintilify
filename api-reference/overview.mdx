Lusha is a RESTful API that lets you query verified contact and company data for prospecting, enrichment, automation, and analytics workflows. All requests are made over HTTPS and return JSON.

**Base URL**

```
https://api.lusha.com
```

## Authentication

All API requests require your API key passed as a request header.

```bash
curl https://api.lusha.com/v2/person \
  -H "api_key: YOUR_API_KEY"
```

Get your API key from the [Lusha dashboard](https://www.lusha.com/). Keep it secret — never expose it in client-side code or public repositories.

## Rate Limiting

The API enforces **25 requests per second** by default. Each response includes headers to help you track your quota:

| Header | Description |
|---|---|
| `X-RateLimit-Limit-day` | Daily request allowance |
| `X-RateLimit-Remaining-day` | Requests remaining today |
| `X-RateLimit-Limit-minute` | Per-minute request allowance |
| `X-RateLimit-Remaining-minute` | Requests remaining this minute |

When you exceed the limit, the API returns `429 Too Many Requests`. Back off and retry after the reset window.

## Error Codes

| Status | Meaning |
|---|---|
| `200` | Success |
| `400` | Bad request — check your parameters |
| `401` | Unauthorized — invalid or missing API key |
| `429` | Rate limit exceeded |
| `5XX` | Server error — retry with exponential backoff |
