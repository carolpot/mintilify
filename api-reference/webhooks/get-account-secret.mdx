Retrieve your current account-level webhook secret. Lusha uses this secret to compute the `X-Lusha-Signature` header included with every webhook delivery. Use the returned secret in your endpoint handler to verify that incoming requests genuinely came from Lusha.

**Endpoint:** `GET https://api.lusha.com/api/account/secret`

## Example request

```bash
curl --request GET \
  --url https://api.lusha.com/api/account/secret \
  --header 'api_key: YOUR_API_KEY'
```

## Example response

```json
{
  "secret": "whsec_abc123def456..."
}
```

## Response fields

<ResponseField name="secret" type="string">
  The current account-level webhook secret. Use this value to verify the `X-Lusha-Signature` header on incoming webhook deliveries.
</ResponseField>

## Using the secret to verify deliveries

When Lusha delivers a webhook, it signs the payload using HMAC-SHA256 with your account secret as the key. To verify the signature in your endpoint handler:

1. Extract the `X-Lusha-Signature` and `X-Lusha-Timestamp` headers from the incoming request.
2. Concatenate the timestamp, a literal `.`, and the JSON-serialized request body.
3. Compute an HMAC-SHA256 digest of the concatenated string using your secret as the key.
4. Use a timing-safe comparison to check that your computed digest matches the value in `X-Lusha-Signature`.

```javascript
const crypto = require('crypto');

function verifySignature(payload, signature, timestamp, secret) {
  const signedPayload = `${timestamp}.${JSON.stringify(payload)}`;
  const expectedSignature = crypto
    .createHmac('sha256', secret)
    .update(signedPayload)
    .digest('hex');

  return crypto.timingSafeEqual(
    Buffer.from(signature),
    Buffer.from(expectedSignature)
  );
}
```

<Warning>
  Always use a timing-safe comparison. Standard string equality is vulnerable to timing attacks.
</Warning>

<Note>
  If you need to rotate your secret, use `POST /api/account/secret/regenerate`. The new secret immediately invalidates the old one and is shown only once in the response.
</Note>
