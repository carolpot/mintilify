sidebarTitle: "POST /prospecting/company/enrich"
description: "Retrieve full company profiles from prospecting search results. Returns firmographics, headcount, revenue, technologies, locations, and classification codes."
openapi: "post /prospecting/company/enrich"
---

After searching for companies with [POST /prospecting/company/search](/api-reference/prospecting/search-companies), use this endpoint to retrieve complete firmographic profiles for up to 100 companies at once. The response includes headcount, revenue ranges, technologies, funding history, intent signals, location details, and more. This is **step 3** of the three-step prospecting flow.

## Endpoint

```
POST https://api.lusha.com/prospecting/company/enrich
```

**Authentication:** API key (`ApiKeyAuth`)

---

## Request body

<ParamField body="requestId" type="string" required>
  The `requestId` returned by the Search Companies response for the same session. Example: `"5ad275c8-7dd4-462a-bd45-6bc1970da64e"`
</ParamField>

<ParamField body="companiesIds" type="string[]" required>
  Array of company `id` values from the search results you want to enrich. Minimum 1, maximum 100. Example: `["33222678"]`
</ParamField>

---

## Example request

<CodeGroup>

```bash cURL
curl --request POST \
  --url https://api.lusha.com/prospecting/company/enrich \
  --header 'api_key: YOUR_API_KEY' \
  --header 'Content-Type: application/json' \
  --data '{
    "requestId": "5ad275c8-7dd4-462a-bd45-6bc1970da64e",
    "companiesIds": ["33222678", "1586"]
  }'
```

```json Request body
{
  "requestId": "5ad275c8-7dd4-462a-bd45-6bc1970da64e",
  "companiesIds": ["33222678", "1586"]
}
```

</CodeGroup>

## Response

  The `requestId` used in the request.
</ResponseField>

      Company name. Example: `"Lusha"`
    </ResponseField>

    <ResponseField name="companies.description" type="string">
      Company description.
    </ResponseField>

    <ResponseField name="companies.fqdn" type="string">
      Primary domain. Example: `"www.lusha.com"`
    </ResponseField>

    <ResponseField name="companies.domains" type="object">

    <ResponseField name="companies.social" type="object">
      Social profile URLs.

      <Expandable title="social fields">

    <ResponseField name="companies.funding" type="object">

      Overall hiring surge signals with `newJobsPostedLastWeek`, `historicalAvg`, and `changeRatePercent`.
    </ResponseField>

    <ResponseField name="companies.surgeInHiringByDepartment" type="object[]">
      Department-level hiring surge signals.

    <ResponseField name="companies.surgeInHiringByLocation" type="object[]">
      Location-level hiring surge signals.

    <ResponseField name="companies.headcountIncrease1m" type="object[]">
      1-month headcount increase signals with `baselineEmployeesCount`, `newEmployeesCount`, and `changeRatePercent`.

      Website traffic increase signals with `historicalAvg`, `lastMonthVisits`, and `changeRatePercent`.
    </ResponseField>

    <ResponseField name="companies.itSpendIncrease" type="object[]">
      IT spend increase signals with `estimatedAnnualItSpend` and `changeRatePercent`.
    </ResponseField>

    <ResponseField name="companies.commercialActivityNews" type="object[]">
      Partnership and go-to-market news signals.
    </ResponseField>

    <ResponseField name="companies.financialEventsNews" type="object[]">
      Funding, IPO, and investment news signals.
    </ResponseField>

    <ResponseField name="companies.riskNews" type="object[]">
      Litigation and security risk news signals.
    </ResponseField>
  </Expandable>
</ResponseField>

## Error codes

| Status | Meaning |
|--------|---------|
| `400` | Validation failed — check that `requestId` and `companiesIds` are present and valid. |
| `401` | Unauthorized — your API key is missing or invalid. |
| `429` | Too many requests — you have exceeded the rate limit. |
| `500` | Internal server error — retry with exponential backoff. |
